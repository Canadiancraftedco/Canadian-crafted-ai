# Switched to Groq + Tavily (no Anthropic billing needed)

The discovery agent and Instagram captions no longer use the Anthropic API.
New setup, both free to start:

## 1. Groq (does the writing/reasoning)
1. Go to **console.groq.com** → sign up (free)
2. **API Keys** → Create API Key
3. Copy it — starts with `gsk_...`

Groq's free tier is generous and fast; you likely won't hit limits at this
project's current scale.

## 2. Tavily (does the real web search)
1. Go to **tavily.com** → sign up (free, no credit card required)
2. Dashboard → copy your API key — starts with `tvly-...`
3. Free tier: 1,000 searches/month. Discovery runs once/day and uses 1 search
   per run, so this comfortably lasts ~30 months at current usage.

## Environment variables — replace the old one
Remove (or just leave unused, it's harmless): `ANTHROPIC_API_KEY`

Add in Vercel → Settings → Environment Variables:
| Name | Value |
|---|---|
| `GROQ_API_KEY` | from step 1 |
| `TAVILY_API_KEY` | from step 2 |

Redeploy after adding these.

## What changed under the hood
- **Instagram captions**: now generated by Groq's `llama-3.3-70b-versatile`
  model instead of Claude — same prompt/behavior, different model.
- **Product discovery**: now a two-step pipeline —
  1. Tavily runs a real web search for the day's category
  2. Groq reads the actual search result snippets and extracts structured
     product data from them — explicitly instructed to only use what's in
     the snippets, never invent details
- **Image backfill**: unchanged — it never used AI, just direct page fetches.

## One honest tradeoff to know about
Claude's web_search tool could browse and read full pages somewhat flexibly.
Tavily returns search result snippets (title + URL + short excerpt) — a bit
less rich than a full page read. In practice this means: fewer results per
run, and occasionally a product page URL to a search-results/collection page
instead of one exact product. The category rotation + daily cron will
gradually build up a good catalog regardless, but expect slightly more admin
cleanup (rejecting or fixing entries) than before.
